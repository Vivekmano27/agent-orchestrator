## Auth Flow

Per the fullstack-dev skill authentication template:

1. `POST /auth/register` вЂ” hash password, create user, return access + refresh tokens
2. `POST /auth/login` вЂ” verify credentials, return access + refresh tokens
3. `POST /auth/refresh` вЂ” validate refresh token, return new access token
4. `POST /auth/logout` вЂ” invalidate refresh token
5. Protected routes use `Authorization: Bearer <access_token>` header

**Token structure:**
- Access token: short-lived (15min), contains user id + role
- Refresh token: long-lived (7 days), stored in DB, rotated on use
